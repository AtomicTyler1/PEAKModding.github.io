Creating a mod for PEAK is simple using the official BepInEx template. This guide covers installing the template, generating a project, and configuring it for development.

## Installing

Before creating a mod, you must install the .NET template. This only needs to be done once.

:::note
You must use .NET SDK 8 or newer. Older versions are unsupported.
:::

### From NuGet (Recommended)

Install the template by running:

```bash
dotnet new install PEAKModding.BepInExTemplate
```

### Manual Install

If you're contributing to the template or prefer manual installation:

1. Clone or download this repository
2. Open a terminal in the root of the project
3. Run:

```bash
dotnet new install .
```

To update:  
```bash
dotnet new install . --force
```

To uninstall:  
```bash
dotnet new uninstall .
```

Once installed, the template will be available as `PEAK BepInEx Plugin`.

## Creating a New Mod

Run the following command in your PEAK modding directory:

```bash
dotnet new peakmod --name ModName --guid com.github.YourAccount.ModName
```

This will create a new folder called `ModName` with your mod's project.

:::tip
Use `--no-tutorial` to skip tutorial comments in the generated code.
:::

## Project Structure

An example structure after generating a mod:

```bash
~/Workspace/PEAK$ dotnet new peakmod --name PeakMod --guid com.github.PEAKModding.PeakMod
~/Workspace/PEAK$ cd PeakMod/
~/Workspace/PEAK/PeakMod$ tree
.
├── Config.Build.user.props.template
├── Directory.Build.props
├── Directory.Build.targets
├── LICENSE
├── PeakMod.sln
├── NuGet.Config
├── README.md
└── src
    └── PeakMod
        ├── PeakMod.csproj
        └── Plugin.cs
```

Key points:

- Your mod code is in `src/PeakMod/`
- `Directory.Build.*` files define shared build settings
- `PeakMod.sln` is your project solution

## Configuring Your Game Path

To allow the build system to deploy directly to your PEAK install:

1. Copy `Config.Build.user.props.template` → `Config.Build.user.props`
2. Open it in a text editor
3. Replace the placeholder with your game path:

```xml
<GameDir>C:/Program Files (x86)/Steam/steamapps/common/PEAK</GameDir>
```

Save the file. Your project can now build directly into the game folder.

## Manual Build Commands

If you're not using Visual Studio or VS Code, you can build from the terminal.

:::note
These commands require a valid `Config.Build.user.props` file.
:::

### Deploy to Game

```bash
dotnet build -p:DeployToProd=true
```

### Build for Release

```bash
dotnet build -p:BuildStaging=true
```

This creates a zip in the `artifacts/` folder for sharing or GitHub releases.

:::warning
The staging build requires a `LICENSE` file in your project root (alongside the `.sln`).
:::
